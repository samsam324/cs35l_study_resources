Security & Auth

CIA: Confidentiality·Integrity·Availability.
AuthN ≠ AuthZ: who you are vs what you may do.
Principles: Least Privilege·Zero Trust (sanitize all input)·Defense in Depth·Open Design (no security by obscurity)·Fail Securely·Trust Boundary.
Threat model: who?·what?·resources?·incentive? Plan: Security model·Threat model·Attack surface·Protections.
Common attacks:
• SQLi: ' OR ''=''. FIX: parameterized queries ?/@1; never concat.
• XSS: inject <script> into trusted page. FIX: escape output, CSP, HttpOnly cookies.
• CSRF: ride logged-in session. FIX: CSRF tokens, SameSite cookies.
• Path traversal ../; command injection; RCE; SSRF; DoS.
Crypto: Symmetric AES (same key). Asymm/PK RSA/ECC (pub encrypt, priv decrypt). Hash SHA-256 (1-way). Sign: priv signs hash; pub verifies. Salt passwords + slow hash (bcrypt/scrypt/argon2).
HTTPS=HTTP+TLS. TLS handshake: cert (CA-signed), key exchange, symmetric session key.
Auth schemes:
• Session cookies: server stores session ID; HttpOnly+Secure+SameSite.
• JWT: stateless; header.payload.signature (base64url). Store in HttpOnly cookie (XSS-safe) NOT localStorage.
• OAuth2: delegated auth (3rd-party login). OIDC=OAuth+identity layer.
Password storage: never plain or unsalted hash; use bcrypt/argon2.
OWASP top: broken access ctrl·crypto failures·injection·insecure design·misconfig·vulnerable deps·auth fail·SSRF.

Networking / Client-Server

Architecture: client-server (centralized) vs P2P (decentralized; both producer+consumer). Hybrid: Zoom.
Throughput=vol/time; latency=time/req. Scale: vertical (bigger) vs horizontal (more).
TCP/IP stack (top→bottom): 1 Application (HTTP/HTTPS/SSH/DNS/FTP/SMTP) 2 Transport (TCP/UDP) 3 Internet (IPv4/IPv6) 4 Link (Ethernet/WiFi/MAC).
TCP reliable, ordered, 3-way handshake SYN→SYN-ACK→ACK, teardown FIN. UDP fire-and-forget; for streaming/games/DNS.
IP: v4 32-bit 4 octets; v6 128-bit. 127.0.0.1=localhost. Ports 0-65535; well-known <1024 (80,443,22,25,53).
DNS name→IP. HTTPS=443; HTTP=80.
HTTP: stateless req/resp. Verbs: GET(safe,idemp,no body)·POST(create, NOT idemp)·PUT(replace, idemp)·PATCH(partial)·DELETE(idemp)·HEAD·OPTIONS(CORS).
Safe=read-only. Idempotent=same effect if repeated.
Status: 1xx info·2xx 200 OK, 201 Created, 204 No Content·3xx 301 perm, 302 found, 304 NotMod·4xx 400 Bad, 401 Unauth, 403 Forbid, 404 NotFound, 409 Conflict, 429 Rate·5xx 500 Err, 502 BadGw, 503 Unavail, 504 GwTimeout.
Headers: Content-Type (application/json), Authorization, Cookie, CORS (Access-Control-Allow-Origin), Cache-Control.
URL: scheme://host:port/path?query#frag.
WebSocket: full-duplex over TCP after HTTP upgrade.
CORS: browser same-origin policy; server allows cross-origin via headers; preflight OPTIONS for non-simple.

Data Management

DBMS handles concurrency, recovery, indexing, optimization. SQL=declarative (WHAT not HOW).
ERD → Relations: entity→table, attr→column, relation→FK or junction table (M:N).
Keys: Primary (unique, NOT NULL); Foreign (refs PK, integrity); Composite; Surrogate.
SQL: 
SELECT c, COUNT(*) FROM t
 WHERE x>0 GROUP BY c
 HAVING COUNT(*)>1
 ORDER BY c DESC LIMIT 10;
Joins: INNER (matching) · LEFT/RIGHT (keep one side, null other) · FULL OUTER · CROSS (cartesian).
Normalization: 1NF (atomic) · 2NF (no partial dep) · 3NF (no transitive dep) · BCNF.
ACID (RDBMS): Atomicity · Consistency · Isolation · Durability. Transactions BEGIN/COMMIT/ROLLBACK.
CAP theorem (pick 2): Consistency · Availability · Partition tolerance. RDBMS=CA·Cassandra=AP·MongoDB=CP.
NoSQL types: Document (Mongo)·Key-Value (Redis)·Wide-Column (Cassandra)·Graph (Neo4j). Eventual consistency; horizontal scale; flexible schema.
Index speeds reads, slows writes. B-tree default.
SQLi defense: parameterized queries always.
